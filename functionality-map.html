Session Dots

Colored bars at the top represent each active session. Active and waiting sessions blink. Click one to focus that terminal.

Attention Section

Sessions that need you are expanded with what they're waiting for (tool name, command preview), how long they've waited, and a "Go" button to jump straight to the terminal.

Background Sessions

Working or completed sessions shown in compact one-line rows with state, token count (e.g. "2.6k"), and time since last event.

Recent Timeline

Last 5 notifications with style dot, title, project name, and relative timestamp. History capped at 50 items.

Session States
Each session transitions between these states based on hook events
Waiting Permission P100
Error P90
Waiting Input P80
Active P50
Idle P10
Stopped P0

The highest-priority session state drives the menu bar icon. Permission and input states never auto-dismiss. Idle and error auto-dismiss after your configured delay.

🧠
Smart Suppression
Prevents notification fatigue without missing anything important

Critical events always shown

Permission requests, errors, and waiting-for-input are never suppressed. They reach you immediately.

Rapid-fire batching

3+ events from the same session in 10 seconds? Idle completions get batched instead of spamming you one by one.

Escalation after 2 minutes

If a permission request stays unresolved for 120+ seconds, the notification escalates with "Still waiting" to re-alert you.

Stale popup auto-dismiss

When a session leaves a waiting state (you handled the permission), any lingering popup for it is automatically dismissed.

Popup deduplication

If a session already has a waiting popup visible, duplicate waiting popups for the same session are skipped.

🔌
Hook System
Auto-installs into ~/.claude/settings.json on launch

6 hook events

SessionStart, Stop, StopFailure, Notification, PermissionRequest, SessionEnd. Each fires an HTTP POST to the local server.

HookInstaller.swift

Timestamped backups

Before modifying settings.json, creates a backup. Keeps max 5, prunes oldest automatically.

Clean uninstall

On quit, removes only Nudgy hooks (preserves your other hooks). Legacy hooks like PreToolUse are cleaned up during install.

🔒
Security & Privacy
Fully local, zero telemetry, defense-in-depth
🔑 Keychain storage 🌐 Localhost only 🔒 Constant-time auth 🛠 No telemetry 🛡 Path traversal protection 🚫 SSRF prevention

Auth tokens and session keys live in macOS Keychain, never in plaintext. HTTP server bound to 127.0.0.1 only. CORS locked to null origin. No external network listeners, no analytics, no remote logging. Open source — inspect every line.

👁
Window Focus
One-click "Go" button jumps to the right terminal

Three strategies tried in order: activate by PID, by known app bundle ID, or any running known terminal as fallback. Walks up the process tree (up to 10 levels) to detect the parent terminal app.

Per-style notification toggles

Turn each notification type on or off independently. Each one has its own sound picker with a play-preview button. "Working" notifications are off by default since they fire on every active session.

nudgy.notify.success nudgy.notify.warning nudgy.notify.question nudgy.notify.error nudgy.notify.info

14 system sounds

Pick from Glass, Ping, Pop, Purr, Basso, Blow, Bottle, Frog, Funk, Hero, Morse, Sosumi, Submarine, or Tink for each notification style. Sounds play on a background dispatch queue so they never block the UI.

nudgy.sound.* nudgy.soundEnabled nudgy.soundVolume

Auto-dismiss delay

Slider from 2 to 30 seconds. Controls how long success and error popups stay visible before fading out. Permission and question popups always persist until you interact.

nudgy.autoDismissDelay

Server port & accessibility

Change the HTTP server port (default 9847, falls back to 9848/9849). Accessibility permission is needed for the "Go" button to focus terminal windows.

nudgy.port

Quick fire buttons

One-click buttons for each of the 5 notification styles. Sends a real popup so you can see exactly how it looks and sounds with your current settings. "Fire All Sequentially" sends all 5 with 0.6s delays.

Custom notification builder

Build a notification from scratch: pick the style, write a title and message, choose a project name, and toggle auto-dismiss. The live preview at the bottom updates as you type.

Live preview
