Critical Issues

5 findings

AppDelegate violates Single Responsibility (8 responsibilities)

CriticalArchitecture

AppDelegate manages HTTP server lifecycle, session management, hook installation, auth tokens, notification pipeline, UI management, event delegation, and notification item creation — all in 272 lines. If any init step fails, the cascade is fragile with no error recovery.

Should extract to: NotificationOrchestrator, NotificationItemFactory, AuthTokenManager, and AppBootstrapper.

AppDelegate.swift:18-79

HTTP server has no connection timeout — resource leak

CriticalSecurity

If a client sends incomplete HTTP headers forever, the connection never closes. Accumulated data grows unbounded until hitting 64KB limit. The recursive receiveData() has no timeout per-connection.

Fix: Add 5-second read timeout using NWConnection.receiveTimeout.

HTTPServer.swift:99-134

Silent failure on hook install and HTTP server start

CriticalUX

If the HTTP server fails to start or hook installation fails, errors are only logged — the user is never notified. The app continues running in a broken state with no hooks and no server, appearing functional.

AppDelegate.swift:51-69

Terminal PID never captured from hook events

CriticalBug

AgentSession.terminalPID and terminalApp are never populated from HookEvent data. This means WindowFocuser.focusSession() can never use Strategy 1 (activate by PID) and always falls back to less reliable bundle ID matching.

SessionManager.swift:29-40

SmartSuppressor ignores user interactions entirely

CriticalBug

recordInteraction() stores timestamp but it is never read. The suppressor completely ignores when the user last dismissed a notification, missing the opportunity to add cooldown-based suppression.

SmartSuppressor.swift:52-53

Notification Pipeline

14 findings

No event deduplication

HighBug

If the same hook event is sent twice (network retry), both create notifications. User sees duplicate "Permission" popups. Need dedup key: session + event name + timestamp within 500ms.

AppDelegate.swift:143-144

HTTP server responds before processing completes

HighBug

Server sends 200 to client immediately, but notification pipeline is still running. Client thinks event processed, but popup may never appear if pipeline fails.

HTTPServer.swift:230

SmartSuppressor batching can misfire across sessions

HighBug

Batches when 3+ events in window AND session is idle. But if user rapidly starts 3 different sessions, each gets batched incorrectly. Should only batch same-session events where state didn't change.

SmartSuppressor.swift:43-46

Escalation only checks first unresolved permission

MediumBug

Uses .first(where:) instead of checking if ANY permission is older than threshold. With 5 unresolved permissions, only the first triggers escalation.

SmartSuppressor.swift:71

Menu bar icon updated 2-3 times per event

MediumPerf

menuBarManager.updateIcon() called multiple times per event processing. Should batch or use reactive observation.

AppDelegate.swift:150, 156, 262

No Content-Length validation before reading body

HighSecurity

HTTP server checks Content-Length AFTER attempting parse. Client can send Content-Length: 1000000 with 100 bytes, causing indefinite wait.

HTTPServer.swift:173-185

No request rate limiting on HTTP server

MediumSecurity

Attacker can flood server with requests, each spawning a new connection. No per-IP throttling or request queue depth limit.

HTTPServer.swift

Query parameter token not URL-decoded

MediumBug

extractQueryParam() doesn't URL-decode values. If token contains %20, auth comparison fails silently.

HTTPServer.swift:252-262

Hook detection is too broad

MediumBug

Detects ANY localhost hook with /event in URL, not just Nudgy's. Could incorrectly remove user's own localhost hooks.

HookInstaller.swift:82-87

Race condition on settings.json

MediumBug

No file locking between read and write. If user edits settings.json while Nudgy installs hooks, updates are lost.

HookInstaller.swift:70-121

Session eviction doesn't prioritize by state

LowPerf

At 100 sessions, evicts oldest by lastEventAt. Could evict stopped sessions first, keeping active/waiting ones alive.

SessionManager.swift:158-164

RingBuffer uses O(n) removeFirst()

MediumPerf

Each append to the 50-element ring buffer calls storage.removeFirst() which is O(n). Should use circular buffer with head/tail indices for O(1).

RingBuffer.swift:12-15

NSSound is deprecated on macOS 12+

LowDeprecation

SoundManager uses NSSound which is deprecated. Should migrate to AVAudioPlayer. Also, single serial queue serializes all playback.

SoundManager.swift:86

WindowFocuser process tree limited to 10 hops

LowPolish

Only walks 10 parent processes. Deep hierarchies (container -> subshell -> subprocess) may truncate. No logging of focus attempt results.

WindowFocuser.swift:61

Architecture & Code Quality

10 findings

AppState array mutations are not atomic

HighBug

Between notifications.insert() and truncation, observers can see 51 items. Also, AppState.sessions has no size cap (SessionManager has 100, but AppState can grow unbounded).

AppState.swift:56-61

Logger lacks structured logging and log levels

Medium

Only log() and error() methods. No debug/warning/info convenience methods. Custom string format instead of JSON makes parsing difficult. Only keeps 2 files (current + .old).

Logger.swift

Concurrency model has unnecessary context switches

MediumPerf

HTTPServer calls delegate on DispatchQueue, then Task { await sessionManager } (actor), then MainActor.run { appState }. Three context hops per event.

AppDelegate.swift:236-238

No retry logic for transient failures

Medium

No retry for network failures, temporary auth failures, or file system errors. A single transient error permanently fails the operation.

Makefile missing pre-build validation and coverage

Low

No checks for signing identity, Apple ID, or Swift version before build. No make lint or make coverage targets. Version hardcoded separately in Makefile.

Makefile

No memory pressure response

LowPerf

Neither AppState nor SessionManager respond to memory warnings. No jettison of cached notifications or sessions under pressure.

Implementation Roadmap

Phase 1 — Critical Fixes

  • Add connection timeout to HTTP server (prevent resource leaks)
  • Surface hook/server errors to user with NSAlert
  • Extract terminal PID from hook events into AgentSession
  • Wire up SmartSuppressor.recordInteraction() to actually influence decisions
  • Fix AppState array mutation atomicity
  • Add AppDelegate and KeychainHelper tests

Phase 2 — Reliability & Polish

  • Add event deduplication in notification pipeline
  • Fix panel width mismatch with popup content views
  • Add accessibility labels and keyboard navigation (ESC to dismiss)
  • Unify color palette into shared design system
  • Fix SmartSuppressor batching to be per-session + state-aware
  • Add Content-Length validation before reading HTTP body
  • Add TranscriptParser and MenuBarManager tests

Phase 3 — Feature Expansion

  • Notification history panel with search/filter
  • Menu bar badge with pending count
  • Keyboard shortcuts (Cmd+D, Cmd+G, Cmd+M)
  • Do Not Disturb / Focus mode integration
  • Per-style auto-dismiss delays
  • Extract NotificationOrchestrator from AppDelegate

Phase 4 — Power Features

  • Multi-agent support (Cursor, Aider, Copilot hooks)
  • Statistics dashboard with Swift Charts
  • Notification grouping / batching UI
  • Custom notification sounds (user-provided audio)
  • Settings export/import
  • CLI testing tool

Phase 5 — System Integration

  • Notification Center widget (WidgetKit)
  • AppleScript / Automator scriptability
  • Webhook/URL actions on notification events
  • Notification persistence across reboots
  • Siri Shortcuts integration
